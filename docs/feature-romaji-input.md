# 機能仕様書: ローマ字→ひらがな変換エンジン

## 1. 概要

Cardputer の 56 キー QWERTY キーボードから日本語（ひらがな）を入力するための、デバイス上で動作するローマ字変換エンジン。外部サーバーや辞書データに依存せず、ファームウェア内で完結する。

## 2. 対象クラス

`PromptInput`（`src/prompt_input.h` / `src/prompt_input.cpp`）

## 3. 入力モード

| モード | 表示 | 動作 |
|---|---|---|
| 日本語モード | `[あ]` | ローマ字入力 → ひらがな自動変換 |
| ASCII モード | `[A]` | 英数字・記号をそのまま入力 |

- **初期状態:** 日本語モード
- **切替:** Tab キーで交互に切り替え（切替時に pending ローマ字をフラッシュ）

## 4. バッファ構造

```
┌──────────────────────┐  ┌────────┐
│  accumulated (確定済)  │  │ romaji │  ← pending（変換待ち）
│  例: "こんにちは"       │  │ 例: "s" │
└──────────────────────┘  └────────┘
              ↑
    LCD 表示: "こんにちはs_"
```

- **accumulated:** 確定済みテキスト（ひらがな + ASCII 混在可）
- **romaji:** 変換待ちのローマ字バッファ（最大 3 文字）
- LCD には `accumulated + romaji + "_"（カーソル）` を表示

## 5. 変換テーブル

### 5.1 対応範囲

| カテゴリ | 例 | エントリ数 |
|---|---|---|
| 母音 | a→あ, i→い, u→う, e→え, o→お | 5 |
| 基本子音+母音 | ka→か, ki→き ... ro→ろ | 45 |
| 濁音 | ga→が, za→ざ, da→だ, ba→ば | 20 |
| 半濁音 | pa→ぱ, pi→ぴ, pu→ぷ, pe→ぺ, po→ぽ | 5 |
| 拗音（3文字） | sha→しゃ, chi→ち, tsu→つ, nya→にゃ 等 | 約30 |
| 外来音 | fa→ふぁ, fi→ふぃ, ja→じゃ 等 | 10 |
| 促音（っ） | bb→っb, kk→っk, ss→っs 等 | 16 |
| ん | nn→ん | 1 |

合計: 約 130 エントリ

### 5.2 変換テーブルの優先順

テーブルは**長いパターンを先に配置**している。マッチは先頭から順に走査し、最初にマッチしたエントリを適用する。

```
促音 (2文字) → 3文字シーケンス → 2文字シーケンス → 1文字母音
```

## 6. 変換アルゴリズム

### 6.1 文字入力時の処理フロー

```
inputChar(c)
  │
  ├─ ASCII モード → accumulated に直接追加 → return
  │
  ├─ 大文字 → 小文字変換
  │
  ├─ 非アルファベット文字
  │     ├─ pending が "n" → accumulated に "ん" を追加
  │     ├─ pending が他 → accumulated にそのまま追加
  │     └─ 文字自体を accumulated に追加 → return
  │
  └─ アルファベット → romaji に追加 → tryConvertRomaji()
```

### 6.2 tryConvertRomaji() の処理

```
tryConvertRomaji()
  │
  ├─ "n" + 子音（a/i/u/e/o/y/n 以外）の特別処理
  │     → accumulated に "ん" を追加、romaji から "n" を除去、継続
  │
  ├─ テーブルから最長一致を検索
  │     ├─ マッチ: 促音の場合（末尾が英字）
  │     │     → ひらがな部分を accumulated に、末尾英字を romaji に残す
  │     ├─ マッチ: 通常
  │     │     → ひらがなを accumulated に、romaji の残りで再帰呼出
  │     └─ マッチなし: 現在の romaji がテーブルの prefix になりうるか？
  │           ├─ なりうる → 保留（次の入力を待つ）
  │           └─ なりえない → romaji の先頭 1 文字を accumulated に移動、残りで再帰
  │
  └─ return
```

### 6.3 "n" の特別処理

`n` は単独でテーブルに含めない（即座に「ん」に変換されると「に」「な」等が入力不可になるため）。

| 入力 | 処理 | 結果 |
|---|---|---|
| n + 母音 (a/i/u/e/o) | 通常の na/ni/nu/ne/no テーブルマッチ | な/に/ぬ/ね/の |
| n + y | ny... テーブルマッチ待ち | にゃ/にゅ/にょ 等 |
| n + n | nn テーブルマッチ | ん |
| n + 他の子音 | 「ん」確定 + 残りの子音で継続変換 | ん + 次のかな |
| n + 非アルファベット | 「ん」確定 + 文字をそのまま追加 | ん + 記号等 |
| n + Enter/Tab | flushPending() で「ん」確定 | ん |

### 6.4 促音（っ）の処理

同一子音の連打で促音を生成する。

| 入力 | 変換 | romaji に残る |
|---|---|---|
| kk | っ | k |
| ss | っ | s |
| tt | っ | t |

テーブルエントリ `{"kk", "っk"}` の結果文字列末尾が英字のため、ひらがな部分（っ）を accumulated に追加し、末尾英字を romaji に戻して次の変換に備える。

## 7. 削除（Backspace）

1. **romaji が非空:** romaji の末尾 1 バイトを削除
2. **romaji が空:** accumulated の末尾 1 文字を削除（UTF-8 マルチバイト対応）

UTF-8 削除ロジック:
```
末尾バイトから先頭方向に走査し、継続バイト（0x80-0xBF）を飛ばして
先頭バイトの位置で切断する
```

## 8. フラッシュ（flushPending）

Enter キー押下（送信時）または Tab キー押下（モード切替時）に呼ばれる。

- romaji が `"n"` → accumulated に「ん」を追加
- romaji が他 → accumulated にそのまま追加（未変換の英字として確定）
- romaji をクリア

## 9. 制約事項

| 項目 | 内容 |
|---|---|
| 漢字変換 | 非対応。ひらがなのみ出力 |
| カタカナ | 非対応 |
| 変換候補選択 | なし（入力と同時に即確定） |
| 入力履歴 | なし |
| カーソル移動 | 非対応（末尾のみ編集可） |
