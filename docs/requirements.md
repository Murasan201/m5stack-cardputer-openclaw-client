# M5Stack Cardputer × OpenClaw 用 要件定義

## 1. はじめに
M5Stack Cardputer を OpenClaw へのプロンプト入力装置として用い、日本語の対話をスムーズに行える UI/制御フローを実装するための要件をまとめます。本システムの目的は、現場やプロトタイピング環境で手元の Cardputer から直接 OpenClaw に問いかけ、回答を Cardputer 上で確認・継続できる体験を提供することです。

## 2. 背景と目的
- Cardputer 上の物理キーとタッチ LCD を使い、PC やスマートフォンに頼らずに OpenClaw へ問い合わせを完結する。  
- 日本語入力を必須とし、入力履歴・対話状態を Cardputer のディスプレイに連動表示。  
- 選択したキーを「対話モード開始キー（プロンプトモード）」として割り当て、キー操作を契機に OpenClaw とのセッションを制御する。

## 3. システムの関係者
| 役割 | 説明 |
| --- | --- |
| Cardputer 利用者 | 物理キー・タッチ操作で会話を始め、入力・送信・結果確認を行う。 |
| OpenClaw | 対話モードイベントを受けて推論を実行し、応答テキストを Cardputer へ返す。 |
| ネットワーク環境 | Cardputer と OpenClaw 間の通信を仲介。接続不良時のフィードバックも必要。 |

## 4. 用語定義
- プロンプトモードキー：Cardputer 上の物理ボタンのうち 1 つを「対話モード開始」として割り当てたもの。  
- プロンプトモード：Cardputer 側で OpenClaw との対話準備状態。LCD に専用メッセージを表示。  
- 日本語入力エリア：Cardputer の画面／キー操作で入力する文字列をリアルタイム表示。  
- 送信ボタン：入力完了後に押すことで OpenClaw へテキストを送るキー／タッチ領域。  

## 5. 機能要件
### 5.1 対話モードの起動
1. 指定されたプロンプトモードキーが押下されたタイミングで OpenClaw 側へ「対話モード開始」イベントを送信。  
2. Cardputer の LCD には「OpenClaw 対話モード：ご質問ください」などの開始メッセージを表示し、現在のモードをユーザーに明示。  
3. 対話モード中は、通常の操作とは別の入力／表示フローに切り替える。  

### 5.2 日本語入力対応
- タッチキーボード、物理キー、またはログから日本語を生成できる簡易 IME を用意し、Cardputer 側で文字列を組み立てる。  
- 入力された文字は即時 LCD に反映され、文字列のカーソル位置と更新状態がわかるように表示。  
- 変換候補や確定処理を必要とする場合は、Cardputer での操作/キーと連携できるようレイヤー構成で定義。  

### 5.3 送信操作
- 日本語入力終了後に専用の「送信ボタン」を押すと、Cardputer は蓄積されたテキストを OpenClaw に送る。  
- 送信時には送信中インジケータ（例：LCD 上のスピナー、進行バー）を表示し、処理中であることを明示。  
- OpenClaw には「ユーザー入力」イベントとしてプロンプトを渡し、推論を直ちに実行させる。  

### 5.4 OpenClaw からの応答受信
- 応答が返ってきたら Cardputer の LCD に（1）相手の回答（2）対話の継続ボタン／入力ヒント をそれぞれ表示。  
- 応答テキストの長さに応じてスクロールやページ送りを実装し、画面上に収まるよう制御。  
- 応答受信直後に Cardputer 側で次の入力を可能にし、必要に応じて「会話を続ける」表示を行う。  

### 5.5 セッション状態管理
| 状態 | 説明 |
| --- | --- |
| 待機 | プロンプトキー未押下。LCD は通常情報または時刻を表示。 |
| 対話モード | キー押下後。専用メッセージ＋入力エリア＋送信ボタンを表示。 |
| 送信中 | 送信ボタン押下後。OpenClaw 応答待ち。 |
| 応答表示 | OpenClaw からの返信を LCD に表示。 |
| エラー | 通信失敗や応答なし。エラーメッセージ＋リトライ指示を表示。 |

- Cardputer 側は状態遷移を内部で追跡し、OpenClaw 側に状態イベント（開始・送信・応答受信・終了）を送出。  
- 対話モードが終了した際には、LCD を既定の待機表示に戻すとともに、OpenClaw にセッション終了イベントを通知。  

## 6. 非機能要件
- 日本語表示：Cardputer のフォントを事前に用意し、文字化けなく日本語を表示。  
- 接続監視：Wi-Fi 接続が切れた場合、画面上に明示的な再接続案内を表示。  
- セキュリティ：OpenClaw との通信には TLS 等の暗号化チャネルを使い、認証キーを渡す。  

## 7. ハードウェア／ソフト構成
- **表示**：Cardputer の 320×240 TFT を活用し、上下三分割で「ステータス」「入力欄」「応答欄」を割り当てる。  
- **入力**：プロンプトモードキー（側面または背面のボタン）を 1 つ割り当て。送信・キャンセル・日本語変換などは画面内のタッチ領域または追加キーに配置。  
- **通信**：Cardputer 側で Wi-Fi 接続し、OpenClaw のエンドポイント（ローカル IP／クラウド API）へ HTTPS POST 送信。必要に応じて再送制御。  
- **ファームウェア**：MicroPython/ESP-IDF のアプリを `software/` に配置し、OpenClaw 対応の通信ライブラリと文字入力 UI を備える。  

## 8. 期待するプロンプトフロー
1. ユーザーがプロンプトモードキーを押す。  
2. Cardputer が LCD に「OpenClaw 対話準備中」「任意のキーで日本語入力」等のメッセージを表示し、OpenClaw にモード開始イベント送信。  
3. ユーザーが文字を入力し、LCD にリアルタイム表示。  
4. 送信ボタンを押す → Cardputer は入力文字列を OpenClaw へ渡す。  
5. OpenClaw が推論を行い、回答テキストを Cardputer へ返す。  
6. Cardputer が回答を LCD に表示し、次の入力に備える。  
7. ユーザーが対話を続ける/終了する操作。  

## 9. エラーハンドリング・例外
- 通信失敗時は「再送」ボタンを表示し、Wi-Fi 再接続要求を促す。  
- OpenClaw から異常レスポンス（400/500）を受けた場合は「再試行/対話終了」のボタンを表示。  
- 日本語入力中の強制終了や電源断時に備えて、直前入力をローカルにキャッシュし、再起動後に復元可能に。  

## 10. 拡張・今後の検討
- 1 回の対話履歴をスクロール可能なメッセージログとして記録し、過去の発言を参照。  
- 追加キーを「ショートカット」や「テンプレート応答」にマッピングし、事前準備したプロンプトを選択。  
- StackChan／他デバイスとの連携を見据え、Cardputer 上の対話セッションをオンラインで同期する。
